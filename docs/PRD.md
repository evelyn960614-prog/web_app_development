# 讀書筆記本 (Reading Notebook) 產品需求文件 (PRD)

## 1. 專案概述

### 背景與動機
在閱讀過程中，讀者常會產生許多感悟與想法，但若沒有及時記錄，這些珍貴的情感共鳴往往會隨時間淡忘。傳統的紙本筆記雖然有溫度，但在搜尋、整理與長期保存上較為不便。本專案旨在提供一個數位化的平台，讓愛書人能輕鬆記錄閱讀軌跡，並將零散的心得轉化為個人專屬的知識庫。

### 目標用戶
- **學生**：用於記錄課外閱讀或學術參考書的心得，方便日後撰寫報告或複習。
- **讀者**：一般的愛書人士，希望建立個人的數位藏書清單並分享/收藏閱讀感悟。

### 核心價值主張
「快速數位化，深度留存共鳴」—— 提供簡約流暢的介面，讓使用者能專注於內容紀錄，並透過評分與排行系統，重新發現閱讀的樂趣。

---

## 2. 功能需求

以下為系統的主要功能及其對應的使用者故事：

1. **記錄書名與基本資訊**
   - 作為 **讀者**，我希望 **能輸入書名、作者與閱讀日期**，以便 **建立完整的閱讀清單**。
2. **撰寫閱讀心得**
   - 作為 **讀者**，我希望 **能為每本書撰寫詳細的心得與感悟**，以便 **深入記錄閱讀過程中的情感共鳴**。
3. **書籍評分系統**
   - 作為 **讀者**，我希望 **能為讀過的書進行 1-5 星的評分**，以便 **直觀地表達我對該書的推薦程度**。
4. **搜尋功能**
   - 作為 **讀者**，我希望 **能透過書名或關鍵字進行搜尋**，以便 **快速找到過去記錄過的特定書籍與心得**。
5. **閱讀排行榜**
   - 作為 **讀者**，我希望 **系統能根據評分產生排行**，以便 **回顧自己的年度最愛或高分推薦書單**。

---

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Python Flask。
- **前端模板**：使用 Jinja2 渲染動態網頁。
- **資料庫**：使用 SQLite 儲存書籍、心得與評分資料。

### 效能與安全考量
- **響應速度**：頁面載入與搜尋反應應在 1 秒內完成。
- **資料完整性**：確保在刪除或修改記錄時，資料庫能正確同步。
- **安全性**：基本的輸入驗證，防止 SQL 注入攻擊。

## 4. MVP 範圍

為了確保專案能如期完成，我們將功能分為三個層級：

| 優先等級 | 功能描述 |
| :--- | :--- |
| **Must Have (核心)** | 書籍新增、心得撰寫、評分功能、書籍清單顯示。 |
| **Should Have (重要)** | 書籍搜尋功能、清單排序（依日期）。 |
| **Nice to Have (加分)** | 閱讀排行榜、封面圖片上傳、分類標籤、深色模式切換。 |
